API Templates — Qualquer API vira Tool via JSON + Skill

Mesmo padrão dos provider_templates: define no JSON → botão na UI → tool pro agente → skill ensina

🎯 O conceito

3 camadas que se completam
┌──────────────────────────────────────────────────┐
│ api_templates (JSON)                             │
│ Define: URL, auth, endpoints, parâmetros, schema  │
│ → Botão aparece na UI de APIs                     │
└────────────────┬─────────────────────────────────┘
                 │ alimenta
┌────────────────▼─────────────────────────────────┐
│ APICallTool (AgentTool genérica)              │
│ Recebe: api_name, endpoint, params                │
│ → Constrói HTTP request do JSON                   │
│ → Retorna response pro modelo                     │
└────────────────┬─────────────────────────────────┘
                 │ ensinada por
┌────────────────▼─────────────────────────────────┐
│ skill (prompt no JSON)                         │
│ Ensina o modelo: quando usar, como chamar,        │
│ o que esperar de resposta, exemplos               │
└──────────────────────────────────────────────────┘

🔧 APICallTool — a tool genérica

Uma única AgentTool resolve qualquer API definida no JSON
struct APICallTool: AgentTool {
    let name = "api_call"
    let description = "Chama uma API externa registrada."
    let parameters: AIToolParameters = [
        "api_name": { type: "string", description: "Nome da API (ex: github, linear)" },
        "endpoint":  { type: "string", description: "Nome do endpoint" },
        "params":    { type: "object", description: "Parâmetros do endpoint" }
    ]

    func execute(_ input: ToolInput, context: ToolContext) async throws -> ToolResult {
        // 1. Lookup: api_templates[input.api_name] → APITemplate
        guard let api = configProvider.apiTemplate(named: input.apiName) else {
            return .error("API '\(input.apiName)' não encontrada.")
        }

        // 2. Lookup: api.endpoints[input.endpoint] → EndpointDefinition
        guard let endpoint = api.endpoints.first(where: { $0.name == input.endpoint }) else {
            return .error("Endpoint '\(input.endpoint)' não encontrado.")
        }

        // 3. Construir URL (substituir {placeholders})
        let url = endpoint.renderURL(base: api.baseURL, params: input.params)

        // 4. Construir request (method, headers, body, auth)
        let request = endpoint.buildRequest(url: url, params: input.params, auth: api.auth)

        // 5. Executar
        let (data, response) = try await URLSession.shared.data(for: request)

        // 6. Retornar resposta formatada
        return .success(prettyJSON(data))
    }
}

🔄 O modelo em ação

Exemplo de diálogo — o agente usa a API via tool
User: "Quais issues abertas temos no repo wawa-note-ios?"
Agent: "Vou consultar o GitHub..."
→ api_call(api_name: "github", endpoint: "list_issues", params: {owner: "...", repo: "wawa-note-ios", state: "open"})
Tool Result: [{ title: "Fix transcription bug", labels: ["bug"], ... }, { title: "Add dark mode", labels: ["enhancement"], ... }]
Agent: "Encontrei 2 issues abertas:
1. Fix transcription bug [bug]
2. Add dark mode [enhancement]
Quer que eu crie uma branch pra alguma delas?"

📐 Estrutura final do JSON — providers + apis

ai_config.json completo
{
  "provider_templates": { ... },    ← AI Providers (OpenAI, Anthropic, Ollama...)
  "api_templates":     { ... },    ← APIs externas (GitHub, Linear, Notion...)
  "model_policy":      { ... },    ← Tiers + budget + feature mapping
  "features":          { ... },    ← Temperature, maxTokens, prompts
  "agent_modes":       { ... },    ← Tools + system prompts por modo
  "providers":         { ... },    ← Instâncias configuradas (com API keys)
  "apis":              { ... }     ← Instâncias configuradas (com API keys)
}

📌 Providers e APIs usam o mesmo padrão: template JSON → botão na UI → tool pro agente → skill ensina. Adicionar GitHub, Linear, Notion, Slack, etc = 1 entrada no JSON. Zero Swift.
