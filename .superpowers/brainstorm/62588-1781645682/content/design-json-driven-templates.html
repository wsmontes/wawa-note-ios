⚙️ Como funciona: ProviderTemplate + UI

O tipo que a UI consome
struct ProviderTemplate: Codable, Identifiable, Sendable {
    var id: String                    // "deepseek"
    var displayName: String           // "DeepSeek"
    var icon: String                  // asset name ou SF Symbol
    var type: ProviderType            // openAICompatible, anthropic, gemini, local
    var baseURL: String               // endpoint base
    var auth: AuthMethod              // api_key_header, api_key_bearer, none
    var authHeader: String?           // "Authorization", "x-api-key"
    var authPrefix: String?           // "Bearer "
    var defaultModels: [String]       // modelos pré-carregados
    var autoDiscover: Bool            // true = escanear rede local
    var discoveryPort: Int?           // porta pra auto-discovery
    var description: String           // tooltip na UI
}

🔄 Fluxo completo: JSON → Botão → Provider vivo

3 camadas, todas data-driven
1. JSON
   provider_templates.deepseek = { type: "openAICompatible", base_url: "...", ... }
                    │
                    ▼
2. UI
   ForEach(configProvider.providerTemplates) → ProviderCard(template)
   → Botão "DeepSeek" aparece automaticamente
                    │
                    ▼  (usuário clica, cola API key, salva)
3. Runtime
   ai_config.json → providers.deepseek = { api_key: "sk-...", enabled: true }
   ProviderResolver.resolve("deepseek") → OpenAICompatibleProvider
   → Pronto pra usar.

Tempo pra adicionar DeepSeek: 30 segundos. Zero código.

🗂️ O que substitui no código atual

Remove

  • ProviderTemplates.swift — hardcoded static accessors
  • ProviderCard.swift — UI hardcoded por provider
  • Switch/case por provider no grid da UI

Substituído por

  • ProviderTemplate — struct Codable
  • JSONConfigProvider.providerTemplates — array do JSON
  • ProviderTemplateCard(template:) — componente genérico data-driven
  • UI renderiza ForEach(templates) — dinâmico por definição

📌 O JSON é a fonte da verdade dos templates. Adicionar entrada = botão novo na UI. Se o tipo for "openAICompatible", zero código. Se for tipo novo (ex: "cohere"), precisa do Provider.swift + 1 case — mas o botão já aparece.
