Design — Seção 3: ModelPolicy + Budget Tracking

Seleção de modelo unificada, budget-aware, configurável por JSON

🔀 Problema atual: Lógica de escolha de modelo espalhada

Hoje: 4 lugares diferentes decidem qual modelo usar
1️⃣ AgentLoop.resolveModel()        ← chat + pipeline
   if mode == .auto: executor 3 iters → advisor
   if budget.isOverBudget: downgrade to executor

2️⃣ BudgetTracker.recommendedTier      ← usa saldo USD
   premium (>50%), standard (25-50%), economy (<25%)

3️⃣ ActiveProviderManager.bestProviderFor() ← prefix heuristics
   gpt- → openAI, claude- → anthropic, gemini- → google

4️⃣ ModelResolver (classe separada)     ← NÃO é usado pelo fluxo principal
   Tiered model resolution com fallbacks. Código morto?

✅ Proposta: ModelPolicy protocol + TieredModelPolicy

Tudo consolidado num protocolo com impl padrão
protocol ModelPolicy: Sendable {
    /// Seleciona o modelo para uma feature, dado o budget atual
    func selectModel(for feature: String, budget: BudgetState) -> ModelSelection
    /// Lista modelos disponíveis no provider ativo
    func availableModels() async -> [String]
    /// Resolve qual provider deve servir um modelo específico
    func providerFor(model: String) -> ProviderType?
}

struct ModelSelection {
    let model: String
    let tier: ModelTier        // .auto, .deep, .fast, .economy
    let provider: ProviderType
    let reason: String         // "feature config", "budget downgrade", "offline fallback"
}

enum ModelTier: String, Codable {
    case auto    // escolhe baseado no budget
    case deep    // modelo mais capaz (reasoning)
    case fast    // modelo mais rápido/barato
    case economy // fallback de custo mínimo
}

📐 TieredModelPolicy — implementação padrão

Lógica de decisão em 3 passos
actor TieredModelPolicy: ModelPolicy {
    let config: AIConfigProvider
    let budget: BudgetTracker
    let network: NetworkMonitor

    func selectModel(for feature: String, budget: BudgetState) -> ModelSelection {
        // Passo 1: Config base — modelo definido em ai_config.json
        let configuredModel = config.modelFor(feature: feature)

        // Passo 2: Budget — downgrade se necessário
        let tier: ModelTier = switch budget.remainingPercent {
            case > 0.50: .deep       // usa o melhor
            case 0.25...0.50: .fast  // modelo rápido
            case < 0.25: .economy    // mínimo
        }

        // Passo 3: Disponibilidade — fallback se offline
        if !network.isConnected {
            return ModelSelection(
                model: config.localFallbackModel(for: feature),
                tier: .economy,
                provider: .local,
                reason: "offline — usando modelo local"
            )
        }

        return ModelSelection(
            model: configuredModel,
            tier: tier,
            provider: config.providerType(for: feature),
            reason: "feature config (budget: \(budget.remainingPercent))"
        )
    }
}
