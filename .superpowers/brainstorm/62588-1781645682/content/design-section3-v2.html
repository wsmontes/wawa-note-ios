Design — Seção 3 (revisada): ModelPolicy 100% JSON-Driven

Nenhuma regra de seleção de modelo no código Swift. Tudo no ai_config.json.

🔑 Princípio: O Swift lê regras do JSON. Não toma decisões.

O TieredModelPolicy vira um motor de rules evaluation. As regras (thresholds, tiers, modelos, fallbacks) são declaradas no JSON.

⚙️ TieredModelPolicy — agora só um motor de rules

Swift: zero decisões hardcoded
actor TieredModelPolicy: ModelPolicy {
    let rules: ModelPolicyRules     // ← carregado do JSON
    let budget: BudgetTracker
    let network: NetworkMonitor

    func selectModel(for feature: String, budget: BudgetState,
                     userTier: String? = nil) -> ModelSelection {

        // 1. Resolve o tier: user override > budget threshold > fallback
        let tierKey: String
        if let forced = userTier, rules.userOverride.enabled {
            tierKey = forced
        } else {
            tierKey = rules.budget.thresholds
                .first(where: { budget.remainingPercent >= $0.minPercent })?
                .tier ?? rules.budget.thresholds.last!.tier
        }

        // 2. Offline? → tier "local"
        let effectiveTier = network.isConnected ? tierKey : "local"

        // 3. Resolve modelo da feature table
        let model = rules.features[feature]?[effectiveTier]
            ?? rules.features["chat"]?[effectiveTier]
            ?? rules.tiers[effectiveTier]?.prefer.first
            ?? "gpt-5.1-mini"

        // 4. Resolve provider pelo prefixo do modelo
        let provider = resolveProvider(for: model)

        return ModelSelection(model: model, tier: effectiveTier,
                              provider: provider, reason: reason)
    }
}

📊 Comparação: Antes vs Depois

O que Antes (hardcoded) Depois (JSON-driven)
Thresholds de budget switch { > 0.50: .premium, ... } thresholds: [{min_percent: 0.50, tier: "deep"}, ...]
Modelos por tier enum ModelTier { case premium, standard, economy } tiers: { "deep": { prefer: [...] }, ... }
Feature → modelo case "analysis": return "claude-sonnet-4-6" features.analysis.deep = "claude-opus-4-8"
Provider por modelo if model.hasPrefix("gpt-") → .openAI Igual (prefix matching é OK — é structural)
Offline fallback Código em ProviderRouter offline_fallback.enabled + tier "local"
User override (Auto/Deep/Fast) AgLoop.resolveModel misturado user_override.enabled + userTier param

📌 Seção 3 revisada: 100% JSON-driven. O Swift só aplica as regras, não as define. Próxima: (4) ProviderResolver simplificado + plano de migração.
