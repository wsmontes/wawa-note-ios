📞 Fluxo de uma Chamada AI Típica

Call Flow: do Service ao Provider
Service Layer (AnalysisService, ChatViewModel, AgentLoop, etc.)
  │
  ├─ 1️⃣ AIConfigService.shared.requestParams(for: "analysis", model: m)
  │     → AIFeatureParams(temperature: 0.4, maxTokens: 4096, contextWindow: 128k, ...)
  │
  ├─ 2️⃣ Constrói AIRequest(model, messages, temperature, maxTokens, responseFormat, tools)
  │
  ├─ 3️⃣ ProviderRouter.resolveActive(context:)
  │     → ActiveProviderManager.getProviderFor(task:) → ProviderRouter.provider(for: config)
  │     → Retorna: any AIProvider (OpenAICompatible / Anthropic / Gemini)
  │
  └─ 4️⃣ provider.send(request) ou provider.sendStreaming(request)
        → HTTP POST → AIResponse / AsyncThrowingStream<AIStreamEvent>

🚨 Problemas encontrados (bypass do AIConfigService)

⚠️ AnalysisService — 2 bypass

  • summarizeChunkWithRetry(): cria AIRequest sem temperature e sem maxTokens
  • tryRetryWithFix(): só seta responseFormat, esquece t° e maxT

⚠️ ContentExtractionService — maxT hardcoded

  • extractTextFromImage(): usa temperature do config mas hardcoda maxTokens: 500

⚠️ ProjectIngestionPipeline — fix retry

  • Retry de JSON malformado não repassa temperature/maxTokens

⚠️ ContentExtractionService — análise de imagem

  • ImageAnalysisService: cap adicional de min(params.maxTokens, 2048) sobre o config
