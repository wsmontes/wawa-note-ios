🔄 Fluxo completo: Add Provider → JSON → Runtime

Tudo converge no ai_config.json
UI Action: User taps "Save Provider"
         │
         ▼
AIConfigPersistence (actor)
  ├─ writeProviderConfig(provider) → atualiza ai_config.json (providers: { ... })
  ├─ writeModelPolicy(policy)      → atualiza ai_config.json (model_policy: { ... })
  ├─ writeFeatureMapping(map)      → atualiza ai_config.json (features: { ... })
  └─ reload()                      → notifica AIService para recarregar config
         │
         ▼
JSONConfigProvider.reload()
  → Relê ai_config.json
  → Atualiza ModelPolicyRules, ProviderConfigs, FeatureConfigs
  → Novos providers/modelos disponíveis IMEDIATAMENTE (sem restart)
         │
         ▼
AIService
  → Próxima chamada send() já usa o novo provider/modelo

Para ADICIONAR um provider NOVO que não existe no template:
  1. User escolhe "Custom" no template picker
  2. Preenche: nome, base URL, API key, path
  3. App testa conexão (fetchModels no endpoint)
  4. User seleciona quais modelos usar
  5. Salva → ai_config.json atualizado → disponível

Para NOVOS MODELOS de um provider existente:
  1. Settings → Provider → "Refresh Models" (refetch do endpoint)
  2. Novos modelos aparecem na lista
  3. User ativa/desativa cada um
  4. User pode assignar a features/tiers via Tier Editor
